# Cargo AI FDocs — Manifest

## Текущее состояние

Утилита уже умеет:
- читать `ai-docs.toml`;
- резолвить версии Rust-зависимостей из `Cargo.lock`;
- скачивать документацию из GitHub (`README.md`, `CHANGELOG.md` и кастомные файлы);
- подрезать `CHANGELOG` под установленную версию;
- сохранять результат в `docs/ai/vendor-docs` в формате, удобном для AI-ассистентов;
- показывать статус синхронизации через `cargo ai-docs status`.

## Фаза 3 — цель

Сделать библиотеку пригодной для повседневного использования в проекте:

1. **Стабильный core пайплайн**
   - консистентная обработка ошибок;
   - единый формат сохранения и индексирования документов;
   - контроль размера файлов и трейсинг операций.

2. **Инкрементальная синхронизация и прозрачность**
   - понятный `status` с сигналами `OK / OUTDATED / MISSING`;
   - предсказуемая структура директорий `<ecosystem>/<name>@<version>`;
   - метаданные источника в каждом скачанном файле.

3. **Подготовка к мульти-экосистемности**
   - модель данных уже содержит `npm` в конфиге и `Ecosystem::Npm` в storage;
   - в этой фазе выравниваем контракты, чтобы в следующей фазе добавить npm-fetcher без ломки API.

## Критерий готовности фазы

Фаза считается завершённой, если:
- проект собирается (`cargo check`);
- `sync` корректно пишет документы по lock-версиям;
- `status` корректно определяет состояние документации;
- ошибки сети/парсинга/файловой системы дают понятные сообщения.
